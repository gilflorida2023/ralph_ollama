Ralph 3.0 - FSM + silent-clone + canonical-doctests

This version fixes the core issues:

1. Added singleton lock (workspace/.ralph.pid) + trap cleanup to prevent multiple ralph loops
2. Silent clone enforcement in _silence_clone() -> ensures capture_output=True is present
3. Canonical sample doctests via CANON_DOCTESTS + regex replacement
4. Improved parse_spec() to handle both "Signature: def ..." and "Task N: ... (main)" patterns
5. agent.py.write_file normalization handles escaped \\nd literals
6. prompt.md doctest rules (single-line, no multiline if)
7. README.md model compatibility section listing 12 models with exact reasons they failed

Key fixes:
- Clone instructions now go directly to workspace/simplesieve, not workspace/workspace/simplesieve
- Git clone calls automatically include capture_output=True so they never leak into model doctests
- Doctest validation no longer fails on malformed prompts
- Clearer "stuck" detection excludes just "not all done" (only stuck when blocked)
- FSM enforcement now reconstructs tasks.py from the snapshot and rejects any extra functions

The harness now reliably runs through all 5 tasks and marks each DONE, with cleaner artifact isolation between runs.
